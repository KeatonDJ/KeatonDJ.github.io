Description

Our project will be a blackjack game and training application.  The player will be able to join a lobby to play against other players in real time or play against a computer.  There is also a training aspect where it will tell you if you made the correct or incorrect decision in real time.  We will also have other helpful bits of information like tips/tricks and an odds table for decision making.

We will be using Java and XML to run the client side of our application.  Spring Framework will be our server side language and we will be using mySQL for Database management.

Languages/Platforms/Libraries

  • Java
  • Spring
  • MySQL
  • Android Studios

Large/Complex

Here is a list of all of the features we plan to implement in Twenty One:

  • The Android mobile card game Twenty One (Blackjack)
  • A playable AI
  • A training feature that helps new players understand the game
  • Support at least 2-player multiplayer
  • User sign-in to see prior game scores, win/loss rates, etc

Creating the game Twenty One is going to take up most of our time as it is the main component to our project. We will need to have images for all of the cards along with the logic for Blackjack.

After this, there needs to be an AI Computer that can play with the user. We were thinking that the AI would be the dealer and not be very smart. If needed, we will add a difficulty feature for the AI Computer.

Next, our team plans to add a statistic system that will display the odds of hitting or staying for each move. We feel this could help with newer players and add more info that can help the user decide what to do.

Lastly, the multiplayer is going to allow for at least 2 players to play against each other. Our ultimate goal is to support 4-player multiplayer.

Our team is going to use Android Studio to create Twenty One, and only one group member has experience using it.. However, none of us have created a mobile app that includes a server-side and database. We all know how to code in Java, but haven’t used Java Spring before to handle server-side interactions. As for MySQL, no one has experience using it or how to handle the database.
