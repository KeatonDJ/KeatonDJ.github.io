ComS 309 - TwentyOne

This project is an android app based on the casino game Blackjack developed in ComS 309 at ISU. This project is intended to be made in 4 development cycles. Each cycle is as follows:
  • First: This first demo covered the project proposal and each group members roles.
  • Second: The second demo required the app to have "round trip" communication. Client to Server, Server to Database, Database to Server, Server to Client.
  • Third: This demo's requirements were that the game's core features should be implemented. In our case this included signing in, joining a lobby, starting a game, playing the game, and finishing the game.
  • Fourth: The final demo will present the secondary features of the app. In this case it will be card backs, player preferences, tuning of the UI, and a chatting feature.

Mars Rover - Object detection

In this portion of the project we were tasked with detecting objects using an IR sensor and PING))) sensor attached to a servo. To accomplish this I had to servo position increment while taking sonar and IR readings. The reason for both is that the sensors we used were good at specific things. The IR sensor was really good at detecting at what angles there was an object and the sonar sensor was really good at reading the distance of that object.

After gathering both data sets (180 values for each sensor), we wrote a program that looked at the IR sensor for when it saw a drop in the average distance. We then checked the distance data from the PING))) sensor and calculated how far away it was.

Finally, we needed to know how wide the object was. This was accomplished by using the distance value and the angles that this object was present. With this information we could then use the equation

tan(objectAngle/2) = 2*width / distance

to calculate the width of that object by solving for width. Once the program was finished with it's calculations it points at the object that was smallest.

Mars Rover - Final Design

For the final demonstration of this project my group was placed in at an isolated computer blind to the testing field. This field was littered with obstacles, holes, and 4 goal posts which we needed to drive our rover into.

The majority of the work for this project was in the data collection. The IR sensor required the implementation of an ADC function. The servo required a timer in PWM configuration so I could feed it specific values that align with specific angles. The iRobat came with a handy Open Interface that allowed for easy movement controls. Finally we needed WiFi communication to send the bot commands. Each of these programs were written from scratch in C.
